# Интеграция MRZ-модели в веб-сервис

Документ описывает добавленный функционал извлечения машиночитаемой зоны (MRZ)
паспортов с использованием дообученной модели PaddleOCR-VL + LoRA и постобработки
по контрольным цифрам, а также сопутствующие изменения архитектуры.

## Что добавлено

### 1. Пакет `app/mrz/` — самодостаточная логика MRZ
Чистый, тестируемый модуль без зависимости от тяжёлых ML-библиотек:

- `checksum.py` — контрольные цифры TD3 (ICAO 9303) и проверка `validate_td3`.
- `postprocess.py` — коррекция одиночных OCR-ошибок перебором правдоподобных
  замен символов до прохождения контрольных цифр (`postcorrect_line2`).
- `parser.py` — разбор MRZ в нормализованные поля (`parse_td3`), устойчивое
  выделение двух строк из вывода модели (`split_mrz_lines`), нормализация дат и
  кодов государств.
- `localize.py` — локализация полосы MRZ (OpenCV: морфология + выравнивание),
  с запасным нижним кропом.
- `pipeline.py` — оркестрация `run_mrz_pipeline(image, ocr_fn)`: локализация →
  распознавание (внешний callable) → пост-коррекция → проверка → разбор.

### 2. `app/services/mrz_engine.py` — чтение MRZ моделью
Класс `PassportMrzReader` (синглтон, ленивая загрузка) загружает PaddleOCR-VL
через `AutoModelForCausalLM(trust_remote_code=True)` с минимальным шимом
`create_causal_mask` и дообученным LoRA-адаптером, выполняет жадное декодирование
по короткому MRZ-промпту (без подавления повторов — это сохраняет заполнители `<`).

### 3. Паспортная ветка в `PaddleOCRVLEngine` (`app/services/extraction.py`)
В `predict()` для `document_type == PASSPORT` сначала вызывается
`_try_passport_mrz(...)`, который через `run_mrz_pipeline` получает поля,
строит `PassportFields`, проставляет валидацию по контрольным цифрам и
нормализованные сущности. При недоступности адаптера ветка возвращает `None`,
и используется прежняя эвристическая обработка (обратная совместимость сохранена).

### 4. Конфигурация (`app/core/config.py`, `.env.example`)
Новые настройки:
- `passport_mrz_adapter_dir` — каталог LoRA-адаптера (пусто = MRZ-путь выключен);
- `passport_mrz_prompt`, `passport_mrz_max_new_tokens`;
- `mrz_localize`, `mrz_postcorrect` — флаги этапов.

### 5. Фронтенд
В `JobDetailPage.tsx` добавлен индикатор валидности MRZ (зелёный/жёлтый бейдж),
показывающий, прошли ли контрольные цифры и применялась ли пост-коррекция.

## Исправления архитектуры

- **`requirements.txt`: `transformers==5.8.0` → `transformers==4.55.0`.** Прежняя
  версия несовместима с PaddleOCR-VL и искажает прямой проход модели (см. главу 2
  исследования). Добавлены `opencv-python-headless`, `numpy`.
- MRZ-логика вынесена из монолитного `extraction.py` в отдельный тестируемый пакет.

## Как включить MRZ-путь

1. Установить зависимости: `pip install -r requirements.txt`.
2. Скачать/распаковать дообученный адаптер (например, `lora_mrz_v7`).
3. Указать путь в `.env`:
   ```
   PASSPORT_MRZ_ADAPTER_DIR=E:/thesis/outputs/lora_mrz_v7
   ```
4. Запустить backend и worker. При загрузке паспорта движком PaddleOCR-VL
   автоматически используется MRZ-путь.

## Тесты

`backend/tests/test_mrz.py` — 7 тестов пакета `app.mrz` (контрольные суммы,
разбор, пост-коррекция, локализация, пайплайн). Запуск:
```
python tests/test_mrz.py
```
Зависимости тестов: PIL, numpy, opencv (без torch/transformers).
