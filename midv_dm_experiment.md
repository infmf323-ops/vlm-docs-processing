# Эксперимент: классификация подлинности документов (MIDV-DM)

## 1. Постановка задачи

Целью данного эксперимента является бинарная классификация документов:
- **0 — подлинный документ (authentic)**
- **1 — поддельный документ (manipulated)**

Используется датасет **MIDV-DM**, содержащий:
- 900 изображений документов
- 100 подлинных
- 800 модифицированных (разные типы подделок)

Данные разделены по `file_id` на:
- train: 630
- val: 135
- test: 135

Это важно для предотвращения утечки данных.

---

## 2. Проблема датасета

Основная сложность:
- сильный дисбаланс классов (1:8)
- подделки локальные (малые области)
- маленький размер датасета

Это делает задачу сложной для стандартной image classification.

---

## 3. Модель

В эксперименте использовалась модель:

**ConvNeXt-Tiny (pretrained на ImageNet)**

Рассматривались два режима:

### 3.1 Linear Probe
- backbone полностью заморожен
- обучается только классификационная голова

### 3.2 Partial Fine-Tuning
- обучается classifier + последний stage сети
- используется малый learning rate

---

## 4. Обучение

Использовались следующие настройки:

- Loss: `CrossEntropyLoss` с class weights
- Оптимизатор: `AdamW`
- Threshold подбирался на validation
- Метрики:
  - Accuracy
  - Balanced Accuracy
  - F1 macro
  - Recall по каждому классу

## 6. Анализ результатов

### 6.1 Accuracy вводит в заблуждение

Несмотря на более высокую accuracy у fine-tuning (0.80),  
модель фактически игнорирует класс `authentic`.

---

### 6.2 Linear Probe показывает лучший баланс

- значительно лучше распознаёт подлинные документы
- более высокая balanced accuracy
- не смещается в сторону доминирующего класса

---

### 6.3 Fine-tuning ухудшает обобщение

При дообучении:
- модель начинает переадаптироваться под класс `manipulated`
- теряет способность находить `authentic`
- происходит деградация balanced accuracy

---

## 7. Выводы

1. Pretrained признаки ConvNeXt-Tiny уже достаточно информативны для задачи.

2. Linear probing является оптимальным режимом при малом объёме данных.

3. Частичное fine-tuning:
   - увеличивает recall для manipulated
   - но ухудшает баланс классов
   - приводит к переобучению

4. Использование только image-level классификации недостаточно, так как:
   - подделки локальны
   - модель не фокусируется на релевантных областях

---

## 8. Дальнейшие направления

Для улучшения качества модели предлагается:

- использовать **segmentation masks**
- применять **crop подозрительных областей**
- реализовать **multi-task learning (classification + segmentation)**

Это позволит учитывать локальную природу подделок.

---

## 9. Итог

В рамках эксперимента показано, что:

> увеличение сложности модели (ConvNeXt + fine-tuning) не гарантирует улучшение качества.

Ключевым фактором является:
- правильная постановка задачи,
- учет дисбаланса,
- использование локальной информации.
