# Эксперимент: классификация подлинности документов (MIDV-DM)

## 1. Постановка задачи

Целью данного эксперимента является бинарная классификация документов:
- **0 — подлинный документ (authentic)**
- **1 — поддельный документ (manipulated)**

Используется датасет **MIDV-DM**, содержащий:
- 900 изображений документов
- 100 подлинных
- 800 модифицированных (разные типы подделок)

Данные разделены по `file_id` на:
- train: 630
- val: 135
- test: 135

Это важно для предотвращения утечки данных.

---

## 2. Проблема датасета

Основная сложность:
- сильный дисбаланс классов (1:8)
- подделки локальные (малые области)
- маленький размер датасета

---

## 3. Модель

Используется модель:

**ConvNeXt-Tiny (pretrained на ImageNet)**

Рассмотрены три режима:

1. **Out-of-the-box**
2. **Linear Probe**
3. **Partial Fine-Tuning**

---

## 4. Метрики

- Accuracy
- Balanced Accuracy (**основная**)
- F1 macro
- Recall по классам
- Confusion Matrix

---

## 6. Анализ результатов

### Accuracy вводит в заблуждение

Высокая accuracy достигается за счёт предсказания доминирующего класса.

---

### Linear Probe — лучший результат

- лучший баланс классов
- высокая recall для authentic
- максимальная balanced accuracy

---

### Fine-tuning ухудшает баланс

- модель переобучается под класс manipulated
- падает качество распознавания authentic

---

### Out-of-the-box

- модель почти всегда предсказывает manipulated
- практически не распознаёт authentic

---

## 7. Выводы

1. Pretrained признаки уже полезны.
2. Linear probe — оптимальный режим.
3. Fine-tuning ухудшает результат.
4. Простая классификация ограничена.
